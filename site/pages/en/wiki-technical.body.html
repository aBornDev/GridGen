Technical Reference

How GridGen is built

GridGen is a zero-dependency, client-only web app. There is no framework and no backend — the app is just index.html, style.css, and gridgen.js, plus one CDN library for PDF generation. A small static-site generator produces the localized pages.

Generated vs. source. The HTML at the repo root and under wiki/ and nl/ is generated — edit the templates in site/ and re-run node build.js, never the output directly.

Build & languages

Each language is rendered from one set of templates plus a per-locale JSON file. English is emitted at the site root; every other language lives under /<lang>/. The generator also writes the hreflang alternates, the per-language <title>/meta and JSON-LD, a language switcher, and a multilingual sitemap.xml.

Adding a language is a four-step change — content plus a single line of configuration:

  1. Add site/locales/<lang>.json (titles, descriptions, nav labels, schema strings).
  2. Add translated bodies under site/pages/<lang>/.
  3. Register the language code in build.js.
  4. Run node build.js.

State model

All app state lives in one plain object. Every control writes into it, and a single render() function reads from it — a tiny unidirectional data flow with no framework required.

const state = {
  image: null,        // the decoded HTMLImageElement
  imageName: '',      // used to name the exported file
  gridMode: 'cells',  // 'cells' | 'size'
  cols: 10, rows: 10, // for 'cells' mode
  cellSize: 50,       // for 'size' mode (px)
  color: '#000000',
  thickness: 2,
  opacity: 1.0,
  showLabels: true,
  labelStyle: 'chess',     // 'chess' | 'numeric'
  labelPosition: 'outside',// 'outside' | 'inside'
  origin: 'top-left'       // 'top-left' | 'bottom-left'
};

Each input listener mutates one field and calls render(). There is no virtual DOM and no diffing — the canvas is simply redrawn from scratch on every change, which is cheap for a single image.

Rendering pipeline

render() repaints the entire canvas in a fixed order. Drawing back-to-front means later layers naturally sit on top of earlier ones.

  1. Compute geometry — derive cell width/height and column/row counts from the active grid mode.
  2. Size the canvas — image dimensions plus a top and left margin when outside labels are enabled (the gutters that hold the column and row labels).
  3. Paint the background white, then draw the image at the margin offset.
  4. Stroke the grid lines using the chosen color, thickness, and opacity.
  5. Draw labels — either in the outer margin or inside each cell.
  6. Update the info bar with the live cell and image dimensions.

Grid geometry

The two grid modes resolve to the same four values — cols, rows, cellW, cellH — so the rest of the renderer does not care which mode you picked:

if (state.gridMode === 'cells') {
  cols = state.cols;  rows = state.rows;
  cellW = img.width / cols;       // cells may be rectangular
  cellH = img.height / rows;
} else {
  cellW = cellH = state.cellSize; // square cells
  cols = Math.ceil(img.width / cellW);
  rows = Math.ceil(img.height / cellH);
}

When outside labels are on, GridGen reserves a margin sized relative to the image so it scales with resolution:

const baseMargin = Math.max(36, Math.min(img.width, img.height) * 0.045);

Grid lines are clamped to the image bounds with Math.min(i * cellW, img.width) so a partial final cell in by size mode never draws past the edge.

Label algorithm

Column labels in chess style are a base-26 (bijective) conversion — the same scheme spreadsheets use for columns, so after Z comes AA:

function getColumnLabel(i) {
  if (state.labelStyle === 'numeric') return String(i + 1);
  let label = '', n = i;
  while (n >= 0) {
    label = String.fromCharCode(65 + (n % 26)) + label;
    n = Math.floor(n / 26) - 1;
  }
  return label;
}

Row numbering respects the origin setting by flipping the index when the origin is bottom-left:

const rowIdx = state.origin === 'top-left' ? i : rows - 1 - i;

Outside labels are centered in the margin gutters along the top and left. Inside labels are drawn in each cell's top-left corner with a stroked halo behind the fill text so they remain readable over busy image content.

Contrast & legibility

Label color is chosen automatically from the grid color's perceived luminance (the Rec. 601 weighting). Light grid colors get dark labels; dark colors keep their hue:

function isLight(hex) {
  const { r, g, b } = hexToRgb(hex);
  return (0.299 * r + 0.587 * g + 0.114 * b) > 186;
}

For inside labels, the halo flips the other way — a white halo behind dark text, or a black halo behind light text — guaranteeing contrast regardless of what is underneath.

PDF export

Export reuses the already-rendered canvas, so the PDF matches the preview exactly. The canvas is encoded to a PNG data URL and placed on an A4 page whose orientation follows the image's aspect ratio:

const w = canvas.width, h = canvas.height;
const orientation = w > h ? 'landscape' : 'portrait';
const pdf = new jsPDF({ orientation, unit: 'pt', format: 'a4', compress: true });

// available area inside a 24pt page margin
const pageW = pdf.internal.pageSize.getWidth();
const pageH = pdf.internal.pageSize.getHeight();
const margin = 24;
const availW = pageW - 2 * margin, availH = pageH - 2 * margin;

// scale to fit, then center on the page
const scale = Math.min(availW / w, availH / h);
const drawW = w * scale, drawH = h * scale;
const x = (pageW - drawW) / 2, y = (pageH - drawH) / 2;

const imgData = canvas.toDataURL('image/png');
pdf.addImage(imgData, 'PNG', x, y, drawW, drawH);
pdf.save(`${baseName || 'grid-map'}-grid.pdf`);

Because the source canvas is at the image's native pixel dimensions, downscaling to the page keeps grid lines sharp. The output filename is derived from the uploaded file, with its extension stripped and -grid appended.

jsPDF is loaded on demand, not up front. exportPDF() is async and awaits a small loader that injects the CDN <script> the first time it is needed — warmed up in the background as soon as you add an image. A visitor who never exports downloads nothing from the CDN.

let jspdfPromise = null;
function loadJsPDF() {
  if (window.jspdf) return Promise.resolve();
  if (!jspdfPromise) {
    jspdfPromise = new Promise((resolve, reject) => {
      const s = document.createElement('script');
      s.src = JSPDF_SRC;            // the cdnjs UMD bundle
      s.onload = resolve;
      s.onerror = () => { jspdfPromise = null; reject(new Error('Failed to load jsPDF')); };
      document.head.appendChild(s);
    });
  }
  return jspdfPromise;              // memoized; retries after a failure
}

async function exportPDF() {
  await loadJsPDF();
  const { jsPDF } = window.jspdf;
  // … build the PDF (above) …
}

Privacy model

GridGen is client-only. The image is read with the browser's FileReader as a data URL and decoded into an Image element — it is never transmitted anywhere:

const reader = new FileReader();
reader.onload = (ev) => { img.src = ev.target.result; };
reader.readAsDataURL(file);

There is no analytics, no server, and no network request beyond loading the jsPDF library from its CDN — and that happens only on demand. Everything else runs in the page.

Extending GridGen

The single-file-of-logic design makes changes localized. Common extensions:

  • New label styles — add a branch to getColumnLabel() and the label-drawing code.
  • Export formats — the canvas can be turned into PNG/JPEG with canvas.toBlob() for direct image download.
  • Non-uniform grids — the geometry block is the single source of truth for cell sizing.
  • Presets — the flat state object serializes cleanly to JSON for save/load or URL sharing.
