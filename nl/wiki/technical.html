Technische referentie

Hoe GridGen is gebouwd

GridGen is een webapplicatie zonder afhankelijkheden die volledig op de client draait. Er is geen framework en geen backend — de app bestaat alleen uit index.html, style.css en gridgen.js, plus één CDN-bibliotheek voor PDF-generatie. Een kleine statische-sitegenerator maakt de gelokaliseerde pagina's aan.

├── build.js            # static-site generator
└── site/               # generator SOURCE
    ├── layouts/        # base.html
    ├── partials/       # head.html
    ├── locales/        # en.json, nl.json — metadata + schema strings

Gegenereerd versus bron. De HTML in de root van de repository en onder wiki/ en nl/ is gegenereerd — bewerk de sjablonen in site/ en voer node build.js opnieuw uit; bewerk de uitvoer nooit rechtstreeks.

Bouwen & talen

Elke taal wordt gegenereerd uit één set sjablonen plus een per-locale JSON-bestand. Engels wordt gegenereerd in de siteroot; elke andere taal staat onder /<lang>/. De generator schrijft ook de hreflang-alternatieven, de per-taal <title>/meta en JSON-LD, een taalwisselaar en een meertalige sitemap.xml.

Een taal toevoegen is een wijziging in vier stappen — content plus één regel configuratie:

  1. Voeg site/locales/<lang>.json toe (titels, beschrijvingen, navigatielabels, schema-strings).
  2. Voeg vertaalde pagina-inhoud toe onder site/pages/<lang>/.
  3. Registreer de taalcode in build.js.
  4. Voer node build.js uit.

Statusmodel

Alle toepassingsstatus leeft in één enkel object. Elk besturingselement schrijft erin, en één render()-functie leest ervan — een kleine unidirectionele dataflow zonder framework.

const state = {
  image: null,        // the decoded HTMLImageElement
  imageName: '',      // used to name the exported file
  gridMode: 'cells',  // 'cells' | 'size'
  cols: 10, rows: 10, // for 'cells' mode
  cellSize: 50,       // for 'size' mode (px)
  color: '#000000',
  thickness: 2,
  opacity: 1.0,
  showLabels: true,
  labelStyle: 'chess',     // 'chess' | 'numeric'
  labelPosition: 'outside',// 'outside' | 'inside'
  origin: 'top-left'       // 'top-left' | 'bottom-left'
};

Elke invoerluisteraar muteert één veld en roept render() aan. Er is geen virtuele DOM en geen diffing — het canvas wordt bij elke wijziging simpelweg opnieuw getekend, wat weinig rekenkracht kost voor een enkele afbeelding.

Renderpijplijn

render() hertekent het volledige canvas in een vaste volgorde. Door achter-naar-voor te tekenen, liggen latere lagen vanzelf boven eerdere.

  1. Geometrie berekenen — celbreedte/-hoogte en het aantal kolommen/rijen afleiden uit de actieve rastermodus.
  2. Canvas opmaten — afbeeldingsafmetingen plus een marge bovenaan en links wanneer buitenlabels zijn ingeschakeld (de kolommen die de kolom- en rijlabels bevatten).
  3. Achtergrond wit inkleuren, dan de afbeelding tekenen op de margeoffset.
  4. Rasterlijnen trekken met de gekozen kleur, dikte en dekking.
  5. Labels tekenen — ofwel in de buitenmarge ofwel in elk vak.
  6. De infobalk bijwerken met de live cel- en afbeeldingsafmetingen.

Rastergeometrie

De twee rastermodi worden omgezet naar dezelfde vier waarden — cols, rows, cellW, cellH — zodat de rest van de renderer niet weet welke modus is gekozen:

if (state.gridMode === 'cells') {
  cols = state.cols;  rows = state.rows;
  cellW = img.width / cols;       // cells may be rectangular
  cellH = img.height / rows;
} else {
  cellW = cellH = state.cellSize; // square cells
  cols = Math.ceil(img.width / cellW);
  rows = Math.ceil(img.height / cellH);
}

Wanneer buitenlabels zijn ingeschakeld, reserveert GridGen een marge die schaalt met de afbeeldingsresolutie:

const baseMargin = Math.max(36, Math.min(img.width, img.height) * 0.045);

Rasterlijnen worden begrensd tot de afbeeldingsranden met Math.min(i * cellW, img.width), zodat een gedeeltelijk laatste vak in de modus per grootte nooit voorbij de rand wordt getekend.

Labelalgoritme

Kolomlabels in schaakstijl zijn een basis-26-conversie (bijectief) — hetzelfde schema dat spreadsheets gebruiken voor kolommen, zodat na Z de reeks AA volgt:

function getColumnLabel(i) {
  if (state.labelStyle === 'numeric') return String(i + 1);
  let label = '', n = i;
  while (n >= 0) {
    label = String.fromCharCode(65 + (n % 26)) + label;
    n = Math.floor(n / 26) - 1;
  }
  return label;
}

De rijnummering respecteert de instelling van de oorsprong door de index om te keren wanneer de oorsprong linksonder staat:

const rowIdx = state.origin === 'top-left' ? i : rows - 1 - i;

Buiten-labels worden gecentreerd in de margekolommen bovenaan en links. Binnen-labels worden in de linkerbovenhoek van elk vak getekend met een omlijnd halo achter de vultekst, zodat ze leesbaar blijven over drukke afbeeldingsinhoud.

Contrast & leesbaarheid

De labelkleur wordt automatisch bepaald op basis van de waargenomen luminantie van de rasterkleur (de Rec. 601-weging). Lichte rasterkleuren krijgen donkere labels; donkere kleuren behouden hun tint:

function isLight(hex) {
  const { r, g, b } = hexToRgb(hex);
  return (0.299 * r + 0.587 * g + 0.114 * b) > 186;
}

Voor binnenlabels werkt de halo andersom — een witte halo achter donkere tekst of een zwarte halo achter lichte tekst — wat contrast garandeert ongeacht wat eronder zit.

PDF-export

Bij het exporteren wordt het al gerenderde canvas hergebruikt, zodat de PDF exact overeenkomt met het voorbeeldvenster. Het canvas wordt gecodeerd als PNG-data-URL en geplaatst op een A4-pagina waarvan de oriëntatie de beeldverhouding van de afbeelding volgt:

const w = canvas.width, h = canvas.height;
const orientation = w > h ? 'landscape' : 'portrait';
const pdf = new jsPDF({ orientation, unit: 'pt', format: 'a4', compress: true });

// available area inside a 24pt page margin
const pageW = pdf.internal.pageSize.getWidth();
const pageH = pdf.internal.pageSize.getHeight();
const margin = 24;
const availW = pageW - 2 * margin, availH = pageH - 2 * margin;

// scale to fit, then center on the page
const scale = Math.min(availW / w, availH / h);
const drawW = w * scale, drawH = h * scale;
const x = (pageW - drawW) / 2, y = (pageH - drawH) / 2;

const imgData = canvas.toDataURL('image/png');
pdf.addImage(imgData, 'PNG', x, y, drawW, drawH);
pdf.save(`${baseName || 'grid-map'}-grid.pdf`);

Omdat het broncanvas op de native pixelafmetingen van de afbeelding staat, blijven rasterlijnen scherp bij het verkleinen naar de pagina. De uitvoerbestandsnaam wordt afgeleid van het geüploade bestand, waarbij de extensie wordt verwijderd en -grid wordt toegevoegd.

jsPDF wordt op aanvraag geladen, niet vooraf. exportPDF() is async en wacht op een kleine loader die het CDN-<script> injecteert zodra het voor het eerst nodig is — op de achtergrond al voorbereid zodra je een afbeelding toevoegt. Een bezoeker die nooit exporteert, downloadt niets van het CDN.

let jspdfPromise = null;
function loadJsPDF() {
  if (window.jspdf) return Promise.resolve();
  if (!jspdfPromise) {
    jspdfPromise = new Promise((resolve, reject) => {
      const s = document.createElement('script');
      s.src = JSPDF_SRC;            // the cdnjs UMD bundle
      s.onload = resolve;
      s.onerror = () => { jspdfPromise = null; reject(new Error('Failed to load jsPDF')); };
      document.head.appendChild(s);
    });
  }
  return jspdfPromise;              // memoized; retries after a failure
}

async function exportPDF() {
  await loadJsPDF();
  const { jsPDF } = window.jspdf;
  // … build the PDF (above) …
}

Privacymodel

GridGen werkt alleen op de client. De afbeelding wordt gelezen met de FileReader van de browser als een data-URL en gedecodeerd in een Image-element — ze wordt nergens naartoe verzonden:

const reader = new FileReader();
reader.onload = (ev) => { img.src = ev.target.result; };
reader.readAsDataURL(file);

Er zijn geen analyses, geen server en geen netwerkverzoeken buiten het laden van de jsPDF-bibliotheek van zijn CDN — en dat gebeurt alleen op aanvraag. Al het overige draait in de pagina.

GridGen uitbreiden

Het ontwerp met één logicabestand houdt wijzigingen lokaal. Veelvoorkomende uitbreidingen:

  • Nieuwe labelstijlen — voeg een tak toe aan getColumnLabel() en de labeltekencode.
  • Exportformaten — het canvas kan worden omgezet naar PNG/JPEG met canvas.toBlob() voor directe afbeeldingsdownload.
  • Niet-uniforme rasters — het geometrieblok is de enige bron van waarheid voor celgrootte.
  • Voorinstellingen — het platte state-object serialiseert netjes naar JSON voor opslaan/laden of URL-deling.
