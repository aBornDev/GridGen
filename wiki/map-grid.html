Guide

Add a coordinate grid to a map and export a PDF

GridGen lays a coordinate reference grid over a map image and exports a print-ready PDF — handy for orienteering, scouting, and field reference. It overlays a grid on a map you already have; it does not generate the map itself.

Overview

A grid with row/column references makes a map easy to talk about: "meet at C4," "the marker is in square B7." GridGen takes any map image — a screenshot, a scan, or an exported topographic map — draws an even grid on top with chess (A1) or numeric (1,1) labels, and exports an A4 PDF you can print and carry.

Bring your own base map. GridGen works on an image you provide. Grab a screenshot of an online map, scan a paper map, or export one from a mapping service, then drop it into GridGen.

Add a grid, step by step

  1. Get a base map. Use a map image you already have — a screenshot, a scan, or an exported topographic map.
  2. Upload it to GridGen. Drag the map into the preview area or click to select it.
  3. Set the grid. Choose the number of squares, or a fixed cell size in pixels to match the reference grid you need.
  4. Add coordinate labels. Turn on chess (A1) or numeric (1,1) labels so every square has a reference, and place them outside the map for clean margins.
  5. Export as PDF. Download a print-ready PDF to take into the field.

The full control reference lives in the user guide.

Coordinate references

Two labeling styles cover most field needs:

  • Chess (A1) — letters across, numbers down. Quick to read aloud and familiar from board games and battle maps.
  • Numeric (1,1) — numbers on both axes, written column,row. Closer to a coordinate-pair feel.

Set the origin to bottom-left for a Cartesian feel, or top-left to match screen and print conventions. Place labels outside so they sit in clean margins around the map rather than over the terrain.

What this is (and isn't)

GridGen draws a visual reference grid. It is intentionally simple and honest about its scope:

  • ✅ It overlays an even grid with readable coordinate labels on any map image and exports a clean PDF.
  • ✅ The grid is great as a relative reference system for a group — squares everyone can name and point to.
  • ❌ It does not compute real-world coordinates. The squares are not UTM, MGRS, or Lambert grid cells, and they are not georeferenced to the map's projection.
  • ❌ It does not generate or download map data — you supply the map image.

If you need a true coordinate grid tied to UTM/MGRS, use a proper topographic map that already prints those grid lines, and use GridGen only when a simple, printable reference grid is enough.

Use cases

  • Orienteering & scouting — add a shared reference grid to a route or area map and print copies for a group.
  • Tabletop & D&D — turn a map illustration into a battle grid with named squares.
  • Hiking & planning — mark and discuss locations on a printed map by square.
  • Teaching map skills — show how a coordinate grid and references work on a real map.

FAQ

Does GridGen generate a topographic map or pull map data?

No. GridGen overlays a grid on a map image you provide. To get a base map, take a screenshot of an online map, scan a paper map, or export one from a mapping service, then upload it here.

Are the squares real UTM or MGRS coordinates?

No. The grid is a visual reference — evenly spaced squares with A1 / numeric labels. They are not georeferenced to the map's projection, so treat them as a relative reference system, not survey-grade coordinates.

Can I match the grid to an existing map grid?

Partly — use By size mode and adjust the cell size so the overlay lines roughly align with the printed grid on your map image. It is a visual alignment, not a calculated one.

Is it free and private?

Yes. GridGen is free and open source, and your map never leaves your device — all processing happens in your browser.
